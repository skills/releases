### 🔎 Preview

#### Modernize Legacy Code with GitHub Copilot [[skills-dev]](https://github.com/skills-dev/copilot-modernize-my-legacy-code)

> Use Copilot to understand, document, and migrate your project.

In this exercise, you will:

1. Explain the current state of your legacy code.
2. Create a diagram of the data flow in your code.
3. Identify areas of your legacy code that can be improved.
4. Use GitHub Copilot to generate modern code snippets.
5. Replace the old code with the new snippets and test the changes.

### Action Text Variables

The [`skills/action-text-variables`](https://github.com/skills/action-text-variables) Action received a major overhaul and `v3.0.0` release. The original Mustache style templating was replaced with Nunjucks style templating, which adds significantly more functionality and templating options.

See the [v3.0.0 Release Notes](https://github.com/skills/action-text-variables/releases/tag/v3.0.0) for more details and exact code changes.

> [!IMPORTANT]
> 💥 This is a **breaking change** for any templates that use advanced features of Mustache templating. Simple keyword replacement with `{{ my_var }}` is still ok.

### Skills Exercise Development Codespace

The [Skills Exercise Development Codespace](https://github.com/skills/skills-manager) (currently known as "Skills Manager") is a pre-configured multi-workspace VS Code environment designed to for easier authoring and updating of Skills exercises.
The vision is to provide consistent runtime environments, linting settings, test tools, and recommendations for using the [exercise toolkit](https://github.com/skills/exercise-toolkit).

**Status:** `private` - Working towards enough functionality and stability for private preview.

- Add workflow to check all repositories for broken links.
- Switch GitHub MCP server from self-hosted to GitHub hosted.
- Adjust grading documentation to account for exercise steps without a grading check.
