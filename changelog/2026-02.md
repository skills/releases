# February 2026

Welcome to the February 2026 release notes for [GitHub Skills](https://learn.github.com/skills)! 🚀

It's been a robust month focused on reliability, modernization, and some deep improvements across both our exercises and platform infrastructure. Image handling, workflow stability, and instructional clarity have all leveled up. Most notably, we've transitioned to local image assets for exercises, made workflows smarter and more secure, and introduced several vital documentation and maintenance upgrades. Awesome job, team! 💪✨

---

### Highlights

- 🖼️ Massive migration of hosted images to local, in-repo assets across nearly all exercises—improving reliability and offline access.
- ⚙️ Important workflow upgrades with new features, enhanced security (moving from PAT to GitHub App tokens), and better cross-repository support.
- 🛠️ Copilot, Codespaces, CodeQL, and more: Key exercises updated with clearer guides and troubleshooting, enhanced privacy instructions, and compatibility fixes.
- 📚 Improved onboarding, clearer instructional flows, and streamlined publishing tools for exercise content creators.
- 🔒 Improved handling of secrets and supply chain security documentation and tools.
- 🔗 Action allowlists and essential platform tools kept up-to-date for organizational rollout and integration.

## 🧱 Platform

### Foundation updates to our backend tools and workflows, making Skills content more maintainable, secure, and scalable.

#### [Exercise Toolkit](https://github.com/skills/exercise-toolkit) [[📝 changes](https://github.com/skills/exercise-toolkit/commits/main?since=2026-02-01&until=2026-02-28)]

- Automatic replacement of relative image links in step markdown with correct, repository-specific absolute URLs—ensuring images work everywhere (including private repos).
- Switched from `raw.githubusercontent.com` to blob URLs with `?raw=true` for private repo compatibility.
- Workflows simplified, with self-references now using GitHub App tokens instead of personal access tokens for higher security.
- Linting infrastructure and dependencies upgraded, including super-linter to 8.4.0.

- All documentation and assets now reference local images using relative paths for a fully consistent creator experience.
- Improved onboarding flows, project management guides, and tooling integrations with GitHub projects and actions.
- Enhanced VS Code MCP toolset configuration for improved developer productivity.

#### [Skills for EMU](https://github.com/skills/skills-for-emu) [[📝 changes](https://github.com/skills/skills-for-emu/commits/main?since=2026-02-01&until=2026-02-28)]
